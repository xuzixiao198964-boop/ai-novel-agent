# 书架功能测试说明

## 1. 已完成的修改

### 1.1 长篇 200+ 章目录 + 只写前 20 章正文
- **配置**（`app/core/config.py`）：新增 `max_chapters_to_write`，0 表示写满，>0 表示只写前 N 章。
- **Writer**（`app/agents/writer.py`）：当 `MAX_CHAPTERS_TO_WRITE=20` 时，仅生成前 20 章正文（目录仍由 Planner 生成 200+ 章）。
- **运行方式**：在 `backend` 目录执行  
  `python run_long_20ch.py`  
  或设置环境变量后启动流水线：  
  `TOTAL_CHAPTERS=0`、`MAX_CHAPTERS_TO_WRITE=20`、`CHAPTER_RANGE_MIN=200`、`CHAPTER_RANGE_MAX=201`。

### 1.2 书架正文显示与容错
- **字体与背景对比度**（`bookshelf.html` 内联样式）：  
  `.chapter-view .chapter-content` 使用深色背景 `#0d1117`、浅色文字 `#e6edf3`，保证正文可读。
- **正文不可用时的提示**：  
  - 接口返回 200 但 `content` 为空：显示「本章内容暂不可用，可能尚未生成或解析失败。」  
  - 接口报错（如 404/500）：显示「正文返回不可用（网络或服务器错误），请返回目录重试。」并保留「返回目录」链接。
- **成书解析容错**（`app/core/state.py`）：  
  若成书 Markdown 中没有「## 正文」标题，则从第一个「### 第X章」开始解析正文，避免目录有而正文解析不到。

### 1.3 部署
- 部署脚本（`deploy/deploy_ssh.py`）已写入服务器 `.env`：  
  `MAX_CHAPTERS_TO_WRITE=20`、`CHAPTER_RANGE_MIN=200`、`CHAPTER_RANGE_MAX=201`。  
  在服务器上新建并启动的任务会按「200 章目录 + 前 20 章正文」运行。

## 2. 书架全流程自测结果

### 2.1 接口测试（以 http://104.244.90.202:9000 为例）

| 项目 | 接口 | 结果 |
|------|------|------|
| 书架列表 | `GET /api/bookshelf` | 200，返回 `books` 数组，含 task_id、title、chapter_count、created_at |
| 目录 | `GET /api/tasks/{task_id}/novel/toc` | 200，返回 title、toc（章节标签列表）、chapter_count |
| 单章正文 | `GET /api/tasks/{task_id}/novel/chapters/{chapter_index}` | 200，返回 title、content、index、prev_index、next_index、total |

### 2.2 页面流程建议自测步骤

1. **书架入口**  
   打开首页 → 右上角点击「书架」→ 应进入 `bookshelf.html`。

2. **书架列表**  
   - 应显示多本已完结且有成书的作品（卡片：书名、章数、日期）。  
   - 若无书：显示「暂无已发布小说，请先在工作台完成一部作品。」  
   - 若请求失败：显示「加载失败」。

3. **目录页**  
   点击某一本书 → 应显示该书标题和章节目录列表；点击某章进入阅读页。

4. **阅读页**  
   - 正文区域：深色底、浅色字，对比度清晰。  
   - 顶部或底部：上一章 / 当前章/总章数 / 下一章；第一章时「上一章」禁用，最后一章时「下一章」禁用。  
   - 「返回目录」可回到该书目录。

5. **异常情况**  
   - 章节未生成或解析失败：正文区显示「本章内容暂不可用…」。  
   - 接口错误：显示「正文返回不可用…」和「返回目录」链接。

### 2.3 本地 200+20 流水线

- 已在本地用 `run_long_20ch.py` 启动一次「200 章目录 + 前 20 章正文」流水线（后台运行）。  
- 跑完后可在工作台看到新任务；成书生成后该书会出现在书架，可点进目录并阅读前 20 章，验证正文对比度与上下章切换。

## 3. 若书架里仍「看不到正文」或「正文不可用」

- **先确认访问方式**：必须通过 **http://服务器:9000/bookshelf.html** 打开，不能使用 `file://`，否则 `/api` 请求会失败。  
- **再看接口**：浏览器 F12 → 网络，点某一章，看 `GET .../novel/chapters/{index}` 是否 200、响应里 `content` 是否有内容。  
  - 若 404：多为该任务没有成书或该章下标越界。  
  - 若 200 但 `content` 为空：多为成书文件中该章解析不到（已通过「无 ## 正文 时从 ### 开始解析」做了容错，若仍有问题可把该任务的成书文件发一段样例再查）。  
- **字体与背景**：若仍觉得对比度不够，可在 `bookshelf.html` 里改 `.chapter-view .chapter-content` 的 `background` / `color` 为更对比的配色。
