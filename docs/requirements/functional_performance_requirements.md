# AI小说生成Agent系统 - 功能与性能需求文档

## 文档概述

本文档定义了AI小说生成Agent系统的核心功能需求和性能需求，专注于系统在多角度健壮性、功能完整性和性能表现方面的详细要求。

## 1. 系统核心功能需求

### 1.1 多Agent协同工作流

#### 1.1.1 Agent职责与功能

##### **TrendAgent（趋势分析Agent）**
**核心功能**：
- 实时爬取或接入小说平台热门榜单数据
- 分析当前热门题材分布（都市、玄幻、言情、科幻等）
- 统计读者偏好特征（章节长度、更新频率、付费模式）
- 预测市场趋势走向和潜力题材
- 基于历史数据给出章节数建议范围

**输出产物**：
- `trend_analysis.json`：结构化趋势分析报告
  - 热门题材排名及热度指数
  - 读者画像分析（年龄、性别、阅读习惯）
  - 章节数建议范围（最小-推荐-最大）
  - 市场饱和度分析和机会点识别
  - 趋势预测（未来3个月热门方向）

**健壮性设计**：
- 数据缓存机制：24小时内相同查询返回缓存结果
- 降级策略：外部数据源不可用时使用历史基准数据
- 数据验证：对爬取数据进行完整性校验和去重
- 频率控制：避免频繁请求触发反爬机制

##### **StyleAgent（风格解析Agent）**
**核心功能**：
- 深度分析目标小说的语言风格特征
- 提取叙事节奏模式（快慢节奏分布）
- 解析人物对话风格和描写特点
- 识别情感表达方式和修辞手法
- 建立可量化的风格参数体系

**输出产物**：
- `style_parameters.json`：量化风格配置文件
  - 语言复杂度评分（1-10分）
  - 叙事节奏参数（场景切换频率、情节推进速度）
  - 对话占比和风格特征
  - 描写详细程度和感官维度分布
  - 情感强度曲线和转折点模式
  - 修辞手法使用频率统计

**健壮性设计**：
- 多样本分析：基于多章节样本确保风格一致性
- 异常检测：识别风格突变点并标记
- 参数标准化：将主观风格转化为可量化参数
- 模板匹配：支持预定义风格模板快速应用

##### **PlannerAgent（策划Agent） - 详细生成机制**
**核心功能**：
- **故事总纲生成**：
  - 基于趋势分析确定核心题材和主题
  - 设计完整的三幕式故事结构（开端-发展-高潮-结局）
  - 构建主要人物关系网络和成长弧线
  - 设计核心冲突和矛盾解决路径
  - 规划情感曲线和读者情绪引导

- **章节大纲生成**：
  - 将总纲分解为具体章节任务
  - 为每个章节设计核心情节点
  - 规划章节间的过渡和衔接
  - 控制节奏分布（铺垫-发展-小高潮）
  - 确保情节推进的逻辑连贯性

- **人物设定生成**：
  - 创建主要人物的完整档案
  - 设计人物外貌、性格、背景故事
  - 规划人物关系发展和变化
  - 确保人物行为的一致性

**生成机制详细流程**：
1. **输入阶段**：
   - 接收趋势分析结果和风格参数
   - 确定目标章节数和题材方向

2. **创意发散阶段**：
   - 生成多个故事概念草案（3-5个）
   - 评估每个概念的创新性和市场匹配度
   - 选择最优概念进行深化

3. **结构设计阶段**：
   - 应用经典叙事结构模板（英雄之旅、三幕式等）
   - 设计具体的情节转折点和悬念设置
   - 规划情感曲线和读者 engagement 点

4. **细化阶段**：
   - 将整体结构分解为章节级规划
   - 为每个章节设计具体情节点
   - 确保章节间的逻辑衔接和节奏控制

5. **审核-修订循环**：
   - 生成完成后进入审核流程
   - 根据审核结果进行针对性修订
   - 最大支持12次修订循环

**输出产物**：
- `故事总纲.md`：完整的故事框架文档
- `outline.json`：结构化章节大纲数据
- `characters.json`：人物设定数据库
- `planner_log.json`：策划过程详细日志

**健壮性设计**：
- 多版本生成：同时生成多个方案供选择
- 结构验证：检查故事结构的完整性和合理性
- 一致性检查：确保人物、情节、设定的内在一致性
- 复杂度控制：根据章节数调整故事复杂度

##### **WriterAgent（写作Agent） - 详细生成机制**
**核心功能**：
- **章节正文生成**：
  - 根据章节大纲生成详细正文内容
  - 保持与设定风格参数的一致性
  - 实现人物对话的自然性和个性化
  - 提供恰当的环境描写和氛围营造
  - 控制叙事节奏和情节推进速度

- **分批生成策略**：
  - 每批处理3个章节，确保内容连贯性
  - 批次间保持人物和情节的一致性
  - 支持断点续写，中断后可继续生成

**生成机制详细流程**：
1. **上下文准备**：
   - 加载章节大纲和人物设定
   - 获取前文内容作为上下文参考
   - 应用风格参数指导写作

2. **场景构建**：
   - 根据大纲构建具体场景环境
   - 安排人物出场和互动
   - 设置场景氛围和情绪基调

3. **情节展开**：
   - 将大纲情节点展开为具体叙述
   - 控制叙事节奏（快慢结合）
   - 插入适当的悬念和转折

4. **对话生成**：
   - 根据人物性格生成个性化对话
   - 控制对话长度和内容相关性
   - 确保对话推动情节发展

5. **描写补充**：
   - 添加环境、外貌、动作等描写
   - 控制描写详细程度（符合风格参数）
   - 确保描写服务于情节和人物

**输出产物**：
- `ch_01.md` ~ `ch_NN.md`：各章节正文文件
- `writing_log.json`：写作过程详细日志
- `consistency_check.json`：内容一致性检查报告

**健壮性设计**：
- 内容验证：生成后立即进行基础质量检查
- 风格一致性：实时监控并调整以匹配风格参数
- 错误恢复：生成失败时自动重试（最大3次）
- 进度保存：定期保存生成进度，支持中断恢复

##### **PolishAgent（润色Agent）**
**核心功能**：
- **语言优化**：
  - 修正语法错误和表达不当
  - 优化句子结构和段落组织
  - 提升语言流畅度和可读性
  - 统一术语和表达方式

- **风格强化**：
  - 强化与设定风格的一致性
  - 调整修辞手法使用频率
  - 优化情感表达强度
  - 改善叙事节奏感

**润色机制**：
1. **语法检查层**：基础语言错误修正
2. **表达优化层**：句子重组和词汇替换
3. **风格调整层**：强化风格特征一致性
4. **节奏优化层**：调整叙事节奏和段落划分

**输出产物**：
- `ch_01_polished.md` ~ `ch_NN_polished.md`：润色后章节
- `polish_report.json`：润色修改记录和统计

**健壮性设计**：
- 保守修改：确保不改变原文核心内容
- 版本对比：保留修改前后的对比记录
- 质量评估：润色后自动评估改进效果

##### **AuditorAgent（审计Agent） - 详细审核机制**
**核心功能**：

**审核维度体系**：
1. **情节连贯性审核**：
   - 检查情节发展的逻辑合理性
   - 验证前后情节的因果关系
   - 识别情节漏洞和矛盾点
   - 评估悬念设置和解决效果

2. **人物一致性审核**：
   - 验证人物行为与性格设定的一致性
   - 检查人物关系发展的合理性
   - 识别人物形象崩塌或突变
   - 评估人物成长弧线的完整性

3. **逻辑合理性审核**：
   - 检查世界观设定的内部一致性
   - 验证事件发生的概率和合理性
   - 识别常识性错误和逻辑漏洞
   - 评估情节发展的可信度

4. **风格符合度审核**：
   - 量化评估与设定风格的匹配度
   - 检查语言风格的一致性
   - 验证叙事节奏的符合程度
   - 评估情感表达的适当性

5. **语言质量审核**：
   - 检查语法正确性和表达清晰度
   - 评估词汇丰富度和恰当性
   - 识别重复表达和冗余内容
   - 评估整体可读性和流畅度

**审核机制详细流程**：
1. **多轮审核设计**：
   - **第一轮：快速扫描**（识别明显问题）
   - **第二轮：深度分析**（详细检查每个维度）
   - **第三轮：综合评估**（整体质量评分）

2. **问题分类系统**：
   - **严重问题**（必须修复）：情节矛盾、人物崩塌、逻辑硬伤
   - **重要问题**（建议修复）：风格不符、表达不清、节奏失调
   - **一般问题**（可选修复）：语言优化、细节完善、修辞调整

3. **量化评分体系**：
   - 每个维度独立评分（0-10分）
   - 计算加权总分和质量等级
   - 设定通过阈值（如总分≥7.5分）

4. **问题定位和描述**：
   - 精确到具体章节、段落、句子
   - 提供详细的问题描述和示例
   - 给出具体的修改建议和方向

**审核触发条件**：
- **计划性审核**：每个Agent完成后自动触发
- **阈值触发**：当质量评分低于阈值时触发
- **手动触发**：支持用户手动请求审核
- **阶段性审核**：每完成一定章节数进行阶段性审核

**输出产物**：
- `audit_report.json`：结构化审计报告
  - 各维度详细评分和问题列表
  - 问题分类和优先级标注
  - 具体的修改建议和示例
  - 整体质量评估和通过建议
- `quality_metrics.json`：量化质量指标数据
- `audit_log.json`：审核过程详细日志

**健壮性设计**：
- 多模型审核：使用多个审核模型交叉验证
- 置信度评估：对审核结果进行置信度评分
- 争议处理：当审核结果冲突时的仲裁机制
- 学习优化：基于历史审核数据优化审核标准

##### **ReviserAgent（修订Agent） - 详细修订机制**
**核心功能**：

**修订策略体系**：
1. **问题导向修订**：
   - 针对审计报告中的具体问题点
   - 应用相应的修订策略和模板
   - 确保问题得到根本性解决

2. **增量修订原则**：
   - 最小化修改范围，保留有效内容
   - 避免全量重写，提高修订效率
   - 确保修订前后的平滑过渡

3. **多版本管理**：
   - 保留修订前的原始版本
   - 生成多个修订方案供选择
   - 支持修订版本的对比和回滚

**修订机制详细流程**：
1. **问题分析阶段**：
   - 解析审计报告，理解问题本质
   - 对问题进行分类和优先级排序
   - 确定最适合的修订策略

2. **修订方案生成**：
   - 针对每类问题生成修订方案
   - 评估每个方案的改进效果
   - 选择最优方案进行实施

3. **修订实施阶段**：
   - 应用选定的修订方案
   - 确保修订内容与上下文协调
   - 保持整体风格的一致性

4. **修订验证阶段**：
   - 检查修订是否解决了原问题
   - 验证是否引入了新的问题
   - 评估修订后的整体质量提升

**修订类型分类**：
1. **情节修订**：
   - 逻辑漏洞修补
   - 情节矛盾解决
   - 节奏调整优化
   - 悬念强化处理

2. **人物修订**：
   - 行为一致性修复
   - 性格特征强化
   - 关系发展合理化
   - 成长弧线完善

3. **风格修订**：
   - 语言风格统一化
   - 叙事节奏调整
   - 情感表达优化
   - 修辞手法应用

4. **语言修订**：
   - 语法错误修正
   - 表达不清优化
   - 冗余内容删除
   - 可读性提升

**输出产物**：
- `ch_01_revised.md` ~ `ch_NN_revised.md`：修订后章节
- `revision_report.json`：详细修订记录
  - 修订的问题列表和解决方案
  - 修订前后的内容对比
  - 修订效果评估数据
- `revision_log.json`：修订过程详细日志

**健壮性设计**：
- 渐进式修订：分步骤实施，避免一次性大改
- 质量监控：修订过程中实时监控质量变化
- 回退机制：修订效果不佳时支持回退到前一版本
- 学习优化：基于修订效果数据优化修订策略

#### 1.1.2 Agent协同机制

##### **流水线执行模型**
**标准执行流程**：
```
TrendAgent → StyleAgent → PlannerAgent → WriterAgent → PolishAgent → AuditorAgent → ReviserAgent
```

**审核-修订循环流程**：
```
PlannerAgent生成 → AuditorAgent审核 → 审核通过？ → 是 → 进入WriterAgent
                              ↓否
                          ReviserAgent修订 → 重新审核 → 循环计数+1
                              ↓
                        循环计数>12？ → 是 → 标记为失败
                              ↓否
                          继续修订循环
```

**执行控制机制**：
1. **顺序控制**：
   - 严格按定义顺序执行Agent
   - 前一个Agent成功完成后才触发下一个
   - 支持跳过可选Agent（如测试模式下跳过PolishAgent）

2. **并行与串行**：
   - 大部分Agent串行执行确保数据一致性
   - 部分操作支持并行（如多章节同时生成）
   - 并行操作需确保资源隔离和数据同步

3. **流程分支**：
   - 根据审核结果自动选择分支路径
   - 支持条件跳转（如质量达标则跳过修订）
   - 异常情况下的备用流程

##### **状态同步机制**
**状态定义**：
- **就绪（ready）**：Agent已初始化，等待执行
- **运行中（running）**：Agent正在执行任务
- **已完成（completed）**：Agent成功完成任务
- **失败（failed）**：Agent执行失败
- **已停止（stopped）**：用户手动停止Agent

**同步机制**：
1. **中央状态管理器**：
   - 维护所有Agent的当前状态
   - 提供状态查询和更新接口
   - 确保状态变更的原子性

2. **实时状态推送**：
   - 状态变更时立即通知相关组件
   - 支持WebSocket实时推送
   - 前端界面自动更新状态显示

3. **状态持久化**：
   - 每次状态变更立即保存到磁盘
   - 支持系统重启后状态恢复
   - 提供状态变更的历史记录

##### **错误处理机制**
**错误分类**：
1. **可恢复错误**：
   - 网络暂时中断
   - 外部API限流
   - 临时资源不足
   - 处理策略：自动重试（最大3次）

2. **部分失败错误**：
   - 单个章节生成失败
   - 部分审核项目不通过
   - 处理策略：跳过失败部分继续执行

3. **致命错误**：
   - 关键依赖不可用
   - 数据严重损坏
   - 处理策略：停止整个任务并报警

**重试机制**：
1. **指数退避重试**：
   - 第一次重试：等待1秒
   - 第二次重试：等待3秒
   - 第三次重试：等待9秒
   - 最大重试次数：3次

2. **智能重试策略**：
   - 根据错误类型选择重试策略
   - 重试前尝试修复可能的问题
   - 重试失败后提供详细错误分析

**降级策略**：
1. **功能降级**：
   - 复杂功能不可用时使用简化版本
   - 外部服务不可用时使用本地模拟
   - 高质量生成失败时使用基础生成

2. **性能降级**：
   - 资源紧张时降低处理质量
   - 延长处理时间避免系统过载
   - 分批处理减少单次资源需求

##### **进度追踪机制**
**进度定义**：
- **整体进度**：整个任务的完成百分比
- **Agent进度**：单个Agent的内部进度
- **子任务进度**：如章节生成进度

**进度计算**：
1. **权重分配**：
   - 每个Agent分配权重反映其工作量
   - 权重基于历史执行时间统计分析
   - 支持动态调整权重

2. **进度更新**：
   - Agent定期报告进度（如每10%）
   - 关键里程碑自动触发进度更新
   - 用户操作（如暂停）立即更新进度

3. **进度准确性**：
   - 基于实际工作量估算剩余时间
   - 考虑历史执行时间的波动性
   - 提供进度置信度评估

**进度展示**：
1. **可视化展示**：
   - 进度条显示整体完成情况
   - 每个Agent独立进度显示
   - 预计完成时间估算

2. **进度详情**：
   - 当前正在执行的具体操作
   - 已完成的工作项统计
   - 剩余工作项的详细列表

##### **审核-修订循环详细机制**
**循环控制参数**：
1. **最大循环次数**：12次（可配置）
2. **质量提升阈值**：每次修订后质量提升≥0.5分
3. **停滞检测**：连续3次修订质量无显著提升
4. **时间限制**：单个循环最大处理时间（如30分钟）

**循环状态管理**：
1. **循环计数**：记录当前循环次数
2. **质量轨迹**：记录每次审核的质量评分
3. **问题演化**：跟踪问题的解决情况
4. **资源消耗**：监控循环的资源使用情况

**循环退出条件**：
1. **成功退出**：
   - 审核质量达到通过阈值（≥7.5分）
   - 所有严重问题已解决
   - 重要问题解决率达到目标（≥80%）

2. **失败退出**：
   - 达到最大循环次数（12次）
   - 质量停滞超过限制
   - 资源消耗超过预算
   - 用户手动终止

3. **警告退出**：
   - 部分问题无法完全解决
   - 质量接近但未达到阈值
   - 时间或资源限制即将到达

**循环优化策略**：
1. **问题优先级调整**：
   - 根据解决难度调整处理顺序
   - 优先解决影响最大的问题
   - 合并相关问题的修订

2. **修订策略选择**：
   - 根据问题类型选择最佳修订策略
   - 基于历史效果数据优化策略选择
   - 支持多种修订方案并行尝试

3. **学习优化**：
   - 记录每次循环的效果数据
   - 分析修订策略的成功率
   - 优化审核标准和修订策略

**循环监控和干预**：
1. **实时监控**：
   - 循环进度和质量变化
   - 资源使用情况和效率
   - 问题解决率和效果

2. **自动干预**：
   - 检测异常模式自动调整策略
   - 资源超限时自动降级处理
   - 质量停滞时尝试激进修订

3. **人工干预**：
   - 支持用户查看循环详情
   - 允许手动调整审核标准
   - 提供修订建议和指导

### 1.4 质量保障功能

#### 1.4.1 多层次审核体系
**审核架构设计**：
1. **实时审核层**：
   - 在生成过程中实时检查基础问题
   - 防止明显错误进入后续流程
   - 提供即时反馈和修正建议

2. **深度审核层**：
   - 生成完成后进行全面深度审核
   - 覆盖所有质量维度的详细检查
   - 生成结构化审核报告和改进建议

3. **交叉审核层**：
   - 使用多个审核模型交叉验证
   - 减少单一模型的误判和漏判
   - 提高审核结果的可靠性和准确性

**审核标准体系**：
1. **客观标准**：
   - 语法正确性（可自动化检查）
   - 逻辑一致性（可形式化验证）
   - 数据完整性（可程序化验证）

2. **主观标准**：
   - 文学质量评估（需要人工或AI判断）
   - 艺术价值评价（基于审美标准）
   - 读者体验预测（基于市场数据）

3. **量化标准**：
   - 建立可量化的质量评分体系
   - 每个维度独立评分并加权计算
   - 设定明确的通过阈值和等级划分

**审核流程优化**：
1. **优先级审核**：
   - 先审核关键章节和核心情节
   - 优先检查严重问题和硬伤
   - 逐步扩展到细节和优化项

2. **增量审核**：
   - 支持对修订内容的增量审核
   - 只审核发生变化的部分
   - 提高审核效率和针对性

3. **自适应审核**：
   - 根据内容复杂度调整审核深度
   - 基于历史质量数据动态调整标准
   - 优化审核资源的分配和使用

#### 1.4.2 智能修订系统
**修订决策机制**：
1. **问题分析引擎**：
   - 深度分析审核报告中的问题
   - 识别问题的根本原因和影响范围
   - 评估不同修订方案的预期效果

2. **修订策略库**：
   - 建立针对各类问题的修订策略库
   - 每种策略包含适用条件和效果预期
   - 支持策略的组合和定制化调整

3. **效果预测模型**：
   - 预测修订后的质量提升程度
   - 评估修订可能引入的新问题
   - 计算修订的成本效益比

**修订执行机制**：
1. **精准修订**：
   - 最小化修改范围，精准定位问题点
   - 保持上下文连贯性和风格一致性
   - 确保修订内容与整体协调

2. **多方案生成**：
   - 为每个问题生成多个修订方案
   - 评估每个方案的优劣和风险
   - 选择最优方案或组合方案

3. **渐进式修订**：
   - 分步骤实施复杂修订
   - 每步完成后验证效果
   - 确保修订过程的可控性

**修订验证机制**：
1. **即时验证**：
   - 修订完成后立即进行基础验证
   - 检查是否解决了原问题
   - 验证是否保持了基本质量

2. **回归测试**：
   - 确保修订没有破坏其他部分
   - 验证整体一致性和连贯性
   - 检查是否引入了新的问题

3. **效果评估**：
   - 量化评估修订的质量提升
   - 分析修订的成本效益
   - 记录修订经验用于优化

**修订学习优化**：
1. **经验积累**：
   - 记录每次修订的问题和解决方案
   - 分析修订策略的成功率和效果
   - 建立修订知识库和最佳实践

2. **策略优化**：
   - 基于历史数据优化修订策略
   - 识别高效修订模式和技巧
   - 淘汰效果不佳的修订方法

3. **自适应调整**：
   - 根据内容类型调整修订策略
   - 基于作者风格优化修订方式
   - 适应不同质量要求的修订强度
