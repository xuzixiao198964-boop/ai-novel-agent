# AI小说生成Agent系统 - 功能与性能需求文档

## 文档概述

本文档定义了AI小说生成Agent系统的核心功能需求和性能需求，专注于系统在多角度健壮性、功能完整性和性能表现方面的详细要求。

## 1. 系统核心功能需求

### 1.1 多Agent协同工作流

#### 1.1.1 Agent职责与功能
- **TrendAgent（趋势分析Agent）**
  - 功能：分析当前小说市场热门趋势
  - 输出：趋势分析报告，包含热门题材、读者偏好、章节数建议
  - 健壮性：支持趋势数据缓存与复用，避免重复分析

- **StyleAgent（风格解析Agent）**
  - 功能：解析目标小说的写作风格参数
  - 输出：风格参数配置文件，包含语言风格、叙事节奏、人物塑造特点
  - 健壮性：支持多种风格模板，风格参数可配置化

- **PlannerAgent（策划Agent）**
  - 功能：生成故事总纲和详细大纲
  - 输出：
    - 故事总纲（整体框架）
    - 章节大纲（每章详细规划）
    - 人物设定（主要角色档案）
  - 健壮性：支持审核-修订循环，最大12次重试机制

- **WriterAgent（写作Agent）**
  - 功能：根据大纲生成章节正文
  - 输出：各章节完整正文内容
  - 健壮性：支持分批生成，每批3章，避免单次请求过长

- **PolishAgent（润色Agent）**
  - 功能：对生成的正文进行语言润色
  - 输出：润色后的章节正文
  - 健壮性：保留原文核心内容，仅优化表达

- **AuditorAgent（审计Agent）**
  - 功能：质量检查与一致性验证
  - 输出：审计报告，包含问题点和改进建议
  - 健壮性：多维度检查（情节连贯性、人物一致性、逻辑合理性）

- **ReviserAgent（修订Agent）**
  - 功能：根据审计报告进行内容修订
  - 输出：修订后的最终版本
  - 健壮性：支持增量修订，避免全量重写

#### 1.1.2 Agent协同机制
- **流水线执行**：Agent按固定顺序执行，前一个Agent完成后自动触发下一个
- **状态同步**：各Agent状态实时同步到中央状态管理器
- **错误处理**：单个Agent失败不影响整体流水线，支持重试机制
- **进度追踪**：每个Agent的进度百分比实时可查

### 1.2 任务管理系统

#### 1.2.1 任务生命周期管理
- **创建任务**：支持自定义任务名称，自动生成唯一任务ID
- **任务启动**：一键启动完整Agent流水线
- **任务暂停/停止**：支持手动干预任务执行
- **任务状态查询**：实时获取任务整体进度和各Agent状态
- **任务清理**：支持清理已完成或失败的任务

#### 1.2.2 任务数据管理
- **结构化存储**：每个任务独立目录，包含：
  - `meta.json`：任务元数据（状态、创建时间等）
  - `logs/`：各Agent执行日志
  - `output/`：各Agent产出文件
  - `progress/`：进度状态文件
  - `memory/`：长期记忆库（7个真相文件）
- **文件版本管理**：支持产出文件的多版本保存
- **数据导出**：支持任务产出的批量下载

### 1.3 故事生成功能

#### 1.3.1 故事总纲生成
- **核心主题定义**：生成具有吸引力的核心故事主题
- **情节框架构建**：设计完整的故事起承转合结构
- **人物关系网**：创建主要人物及其关系图谱
- **世界观设定**：构建故事发生的背景世界

#### 1.3.2 章节规划功能
- **章节数量自适应**：根据趋势分析建议动态确定章节数
- **章节内容规划**：为每个章节规划核心情节点
- **节奏控制**：合理分配高潮、铺垫、转折的章节位置
- **连续性保障**：确保章节间情节连贯性

#### 1.3.3 正文生成功能
- **风格一致性**：保持与设定风格参数一致的语言风格
- **情节展开**：根据大纲详细展开每个章节的情节
- **对话生成**：生成自然的人物对话
- **环境描写**：提供恰当的场景和环境描写

### 1.4 质量保障功能

#### 1.4.1 自动审核机制
- **内容一致性检查**：验证人物设定、情节逻辑的一致性
- **风格符合度检查**：检查生成内容与设定风格的匹配度
- **逻辑合理性检查**：验证故事情节的逻辑合理性
- **语言质量检查**：检查语法、用词、表达流畅度

#### 1.4.2 修订与优化
- **问题定位**：精确识别需要修订的具体内容
- **针对性修订**：针对不同问题类型采用不同的修订策略
- **修订验证**：修订后重新进行质量检查
- **版本对比**：支持修订前后的内容对比

### 1.5 运行模式管理

#### 1.5.1 测试模式
- **章节数控制**：支持设置测试章节数（如6章、18章等）
- **递增策略**：成功完成后按×3策略递增章节数
  - 初始：6章/18章（可配置）
  - 第2轮：当前章数×3
  - 第3轮：再次×3，直至趋势建议上限
- **快速验证**：通过小规模测试验证系统功能

#### 1.5.2 正式模式
- **趋势驱动**：章节数由趋势分析结果动态决定
- **完整流程**：执行完整的Agent流水线
- **质量优先**：严格执行审核-修订循环确保质量

## 2. 性能需求

### 2.1 响应性能

#### 2.1.1 API响应时间
- **健康检查API**：< 100ms
- **任务状态查询**：< 200ms
- **任务创建**：< 500ms
- **任务启动**：< 1s
- **文件下载**：< 2s（取决于文件大小）

#### 2.1.2 网页加载性能
- **首页加载**：< 1s
- **任务列表加载**：< 1.5s
- **Agent详情加载**：< 2s
- **文件列表加载**：< 2s

### 2.2 处理性能

#### 2.2.1 Agent处理速度
- **TrendAgent**：趋势分析 < 30秒
- **StyleAgent**：风格解析 < 60秒
- **PlannerAgent**：
  - 故事总纲生成：< 3分钟
  - 章节大纲生成：每章 < 30秒
- **WriterAgent**：每章正文生成 < 2分钟
- **PolishAgent**：每章润色 < 1分钟
- **AuditorAgent**：每章审计 < 1.5分钟
- **ReviserAgent**：每章修订 < 2分钟

#### 2.2.2 批量处理能力
- **并发任务支持**：至少支持3个任务同时运行
- **章节批量生成**：支持每批3章的并行生成
- **内存管理**：单个任务内存占用 < 500MB
- **磁盘IO优化**：异步文件写入，避免阻塞主流程

### 2.3 健壮性性能

#### 2.3.1 错误恢复能力
- **LLM API故障**：支持自动重试（最大3次），失败后优雅降级
- **网络中断**：支持断点续传，网络恢复后继续执行
- **磁盘空间不足**：提前检测并预警，避免数据损坏
- **内存溢出**：内存监控与自动清理机制

#### 2.3.2 数据一致性保障
- **任务状态持久化**：每次状态变更立即持久化
- **文件原子写入**：避免写入过程中断导致文件损坏
- **进度同步**：多进程间进度状态实时同步
- **异常回滚**：关键操作失败时自动回滚到上一稳定状态

#### 2.3.3 资源管理
- **CPU使用率**：平均 < 70%，峰值 < 90%
- **内存使用**：有内存泄漏检测和自动清理机制
- **磁盘空间**：定期清理临时文件，保留最近7天数据
- **网络带宽**：优化LLM API调用，减少不必要的数据传输

### 2.4 扩展性性能

#### 2.4.1 负载扩展能力
- **水平扩展**：支持通过增加实例数提升处理能力
- **垂直扩展**：支持通过提升单实例资源配置提升性能
- **任务队列**：支持任务排队机制，避免系统过载

#### 2.4.2 数据量扩展
- **大章节数支持**：支持200+章节的长篇小说生成
- **多任务并发**：支持多个任务并行处理不同小说
- **历史数据积累**：支持长期记忆库的持续积累和复用

## 3. 功能健壮性需求

### 3.1 输入验证与处理

#### 3.1.1 任务创建输入
- **任务名称验证**：非空、长度限制（1-100字符）、字符类型限制
- **章节数验证**：正整数、合理范围（1-500）、与趋势建议的兼容性
- **运行模式验证**：有效的模式选择（test/normal）

#### 3.1.2 API参数验证
- **任务ID验证**：有效的UUID格式、存在性检查
- **页码参数**：正整数、合理范围
- **文件路径**：路径安全性检查，防止目录遍历攻击

### 3.2 异常处理机制

#### 3.2.1 LLM相关异常
- **API调用失败**：重试机制（最大3次），失败后记录详细日志
- **响应解析失败**：JSON解析异常处理，提供有意义的错误信息
- **内容生成失败**：部分失败不影响整体流程，支持跳过继续
- **超时处理**：设置合理的超时时间（LLM调用<30s），超时后重试

#### 3.2.2 系统级异常
- **文件系统异常**：磁盘满、权限不足时的优雅处理
- **内存异常**：内存不足时的资源释放和任务暂停
- **网络异常**：网络中断时的连接恢复机制
- **进程异常**：子进程崩溃时的父进程监控和重启

### 3.3 状态一致性保障

#### 3.3.1 任务状态机
- **状态定义清晰**：ready → running → completed/failed/stopped
- **状态转换验证**：只允许合法的状态转换
- **状态持久化**：每次状态变更立即保存到磁盘
- **状态恢复**：系统重启后能恢复任务到正确状态

#### 3.3.2 进度同步机制
- **实时进度更新**：Agent进度变化实时反映到前端
- **进度持久化**：定期保存进度到文件，避免数据丢失
- **进度一致性**：多Agent间进度逻辑一致性检查
- **进度恢复**：中断后能从最近进度点继续

### 3.4 数据完整性保障

#### 3.4.1 文件完整性
- **文件完整性检查**：重要文件生成后验证完整性
- **文件版本管理**：支持重要文件的多版本保存
- **文件备份机制**：关键文件自动备份
- **文件恢复**：从备份中恢复损坏的文件

#### 3.4.2 数据一致性
- **引用一致性**：文件间的引用关系一致性检查
- **数据去重**：避免重复生成相同内容
- **数据验证**：生成数据的格式和内容验证
- **数据清理**：定期清理无效或过时数据

## 4. 用户体验需求

### 4.1 操作界面功能

#### 4.1.1 任务管理界面
- **任务列表展示**：清晰展示所有任务的状态、创建时间、进度
- **任务操作**：一键创建、启动、停止、删除任务
- **任务筛选**：按状态、时间等条件筛选任务
- **批量操作**：支持批量停止或删除任务

#### 4.1.2 进度监控界面
- **整体进度**：可视化展示整个任务的完成百分比
- **Agent详情**：每个Agent的独立进度和状态
- **实时更新**：进度自动刷新，无需手动刷新页面
- **进度历史**：查看进度的历史变化趋势

#### 4.1.3 日志查看界面
- **日志分级**：支持不同级别的日志（INFO、WARNING、ERROR）
- **日志筛选**：按Agent、时间、级别筛选日志
- **日志搜索**：支持关键词搜索日志内容
- **日志导出**：支持日志的下载和导出

#### 4.1.4 文件管理界面
- **文件树展示**：按任务目录结构展示文件
- **文件预览**：支持文本文件的在线预览
- **文件下载**：支持单个或批量文件下载
- **文件对比**：支持不同版本文件的对比查看

### 4.2 交互反馈

#### 4.2.1 操作反馈
- **即时反馈**：用户操作后立即给出视觉或文字反馈
- **进度提示**：长时间操作显示进度条或等待提示
- **成功确认**：操作成功后的明确确认信息
- **错误提示**：操作失败时的详细错误说明和建议

#### 4.2.2 状态提示
- **系统状态**：清晰显示系统当前运行状态
- **任务状态**：每个任务的当前状态一目了然
- **Agent状态**：各Agent的详细状态和进度
- **异常预警**：系统异常时的醒目提示

### 4.3 可访问性

#### 4.3.1 界面可访问性
- **响应式设计**：支持不同屏幕尺寸的设备访问
- **键盘导航**：支持键盘操作完成主要功能
- **屏幕阅读器**：界面元素有适当的ARIA标签
- **颜色对比度**：满足WCAG AA标准的颜色对比度

#### 4.3.2 操作可访问性
- **操作简化**：常用操作步骤尽量简化
- **操作指引**：新用户有明确的操作指引
- **错误恢复**：操作错误后容易恢复或重试
- **帮助文档**：关键功能有相应的帮助说明

## 5. 集成与接口需求

### 5.1 LLM接口集成

#### 5.1.1 API兼容性
- **OpenAI兼容接口**：支持标准的/v1/chat/completions接口
- **多Provider支持**：支持DeepSeek、OpenAI等不同Provider
- **回退机制**：主接口失败时自动回退到备用接口
- **配置灵活**：API地址、密钥、模型等参数可配置

#### 5.1.2 请求优化
- **请求批处理**：合理批处理LLM请求，减少API调用次数
- **请求超时**：设置合理的请求超时时间
- **请求重试**：网络错误时的自动重试机制
- **请求限流**：避免过快的请求频率触发限流

### 5.2 数据接口

#### 5.2.1 RESTful API
- **标准HTTP方法**：GET、POST、PUT、DELETE的合理使用
- **状态码规范**：使用标准的HTTP状态码
- **响应格式**：统一的JSON响应格式
- **错误处理**：统一的错误响应格式

#### 5.2.2 WebSocket接口
- **实时进度推送**：任务进度变化的实时推送
- **日志实时推送**：日志生成的实时推送
- **状态变更通知**：任务状态变更的实时通知
- **连接管理**：WebSocket连接的建立、维护、关闭

### 5.3 文件接口

#### 5.3.1 文件上传下载
- **文件上传**：支持配置、模板等文件的上传
- **文件下载**：支持任务产出文件的下载
- **文件列表**：支持目录结构的文件列表查询
- **文件信息**：支持文件大小、修改时间等元数据查询

#### 5.3.2 文件格式支持
- **文本格式**：支持.md、.txt、.json等文本格式
- **结构化数据**：支持JSON、YAML等结构化数据格式
- **二进制文件**：支持图片等二进制文件的存储和访问
- **压缩文件**：支持ZIP等压缩格式的打包下载

## 7. 安全需求

### 7.1 访问控制

#### 7.1.1 接口访问控制
- **API认证**：支持API密钥或Token认证
- **权限控制**：不同操作需要不同的权限级别
- **访问日志**：记录所有API访问的详细日志
- **访问限流**：防止恶意请求和DDoS攻击

#### 7.1.2 数据访问控制
- **任务隔离**：不同用户的任务数据完全隔离
- **文件权限**：严格控制文件的读写权限
- **敏感数据保护**：API密钥等敏感数据加密存储
- **数据泄露防护**：防止敏感数据通过日志或错误信息泄露

### 7.2 输入安全

#### 7.2.1 输入验证
- **参数验证**：所有输入参数都进行严格的验证
- **路径安全**：防止目录遍历攻击
- **脚本注入防护**：防止各种脚本注入攻击
- **文件上传安全**：严格验证上传文件的类型和内容

#### 7.2.2 输出安全
- **输出编码**：所有输出都进行适当的编码，防止XSS攻击
