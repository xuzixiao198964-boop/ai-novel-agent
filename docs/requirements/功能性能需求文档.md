# AI小说生成Agent系统 - 功能与性能需求文档

## 文档概述

本文档详细描述AI小说生成Agent系统的功能需求和性能需求，重点关注系统在多角度健壮性方面的要求。文档不包含运行环境、测试、运维、研发计划、邮件短信等非核心内容。

---

## 1. 核心功能需求

### 1.1 多Agent协同工作流

#### 1.1.1 Agent职责与功能
| Agent | 核心功能 | 关键产出 | 质量要求 |
|-------|----------|----------|----------|
| **TrendAgent** | 分析热门小说趋势，确定创作方向 | 趋势分析报告、建议章节数 | 趋势数据准确率≥95%，建议章节数合理 |
| **StyleAgent** | 解析目标风格，生成创作参数 | 风格参数配置、写作模板 | 风格匹配度≥90%，参数可执行性100% |
| **PlannerAgent** | 生成故事策划案和详细大纲 | 策划案、故事总纲、章节大纲 | 故事逻辑连贯性≥95%，章节分配合理 |
| **WriterAgent** | 根据大纲生成章节正文 | 章节草稿（每章） | 内容质量评分≥85分，符合大纲要求 |
| **PolishAgent** | 润色章节内容，提升文笔 | 润色后章节 | 文笔提升度≥20%，保持原意100% |
| **AuditorAgent** | 质量审核，发现问题 | 审核报告、问题清单 | 问题发现率≥95%，误报率≤5% |
| **ReviserAgent** | 根据审核结果修订内容 | 修订后章节 | 问题修复率≥98%，不引入新问题 |

#### 1.1.2 工作流控制
- **顺序执行**: Trend → Style → Planner → Writer → Polish → Auditor → Reviser
- **条件分支**: 审核通过则结束，否则进入修订循环
- **循环控制**: 最大修订次数12次，超过则任务失败
- **状态管理**: 每个Agent独立状态（就绪/运行中/完成/失败）

### 1.2 故事生成功能

#### 1.2.1 故事策划功能
- **总纲生成**: 生成完整故事总纲（核心主题、人物设定、世界观）
- **大纲生成**: 生成详细章节大纲（每章标题、核心情节、关键转折）
- **章节规划**: 合理分配章节内容，确保节奏控制
- **审核机制**: 策划案自动审核，确保逻辑连贯性

#### 1.2.2 内容生成功能
- **章节生成**: 根据大纲生成完整章节内容
- **风格保持**: 保持统一的写作风格和人物性格
- **情节连贯**: 确保章节间情节连贯，伏笔呼应
- **文笔质量**: 生成符合文学标准的文本

#### 1.2.3 质量保障功能
- **自动润色**: 提升文笔、优化表达、修正语法
- **质量审核**: 多维度审核（逻辑、文笔、情节、人物）
- **智能修订**: 根据审核结果自动修订内容
- **迭代优化**: 多轮修订直至质量达标

### 1.3 系统管理功能

#### 1.3.1 任务管理
- **任务创建**: 支持创建新任务，指定任务名称
- **任务控制**: 启动、停止、删除任务
- **状态监控**: 实时显示任务状态和进度
- **历史记录**: 保存任务历史，支持查看和恢复

#### 1.3.2 配置管理
- **运行模式**: 支持测试模式和正式模式
- **章节设置**: 测试模式下可设置测试章节数
- **LLM配置**: 支持多种LLM提供商和模型切换
- **参数调整**: 可调整各Agent的工作参数

#### 1.3.3 文件管理
- **产出文件**: 自动保存各阶段产出文件
- **文件下载**: 支持下载策划案、章节、报告等
- **文件查看**: 在线查看生成的内容
- **版本管理**: 保存修订历史版本

### 1.4 用户界面功能

#### 1.4.1 监控界面
- **总进度面板**: 显示7个Agent的实时状态
- **Agent详情**: 点击查看每个Agent的详细信息和日志
- **进度可视化**: 图形化显示进度百分比
- **实时更新**: 自动刷新状态，无需手动刷新

#### 1.4.2 文件界面
- **文件列表**: 按任务和Agent分类显示文件
- **文件预览**: 支持Markdown、JSON等格式预览
- **文件下载**: 一键下载单个或批量文件
- **文件搜索**: 支持按名称和类型搜索

#### 1.4.3 任务界面
- **任务列表**: 显示所有任务及其状态
- **任务操作**: 创建、选择、启动、停止任务
- **任务筛选**: 按状态筛选任务
- **任务详情**: 查看任务的详细信息和历史

## 3. 健壮性需求

### 3.1 容错能力

#### 3.1.1 错误处理
- **LLM调用失败**: 自动重试3次，重试间隔指数退避
- **网络异常**: 连接超时30秒，读取超时300秒
- **文件操作失败**: 自动重试并记录错误，不影响其他操作
- **内存溢出**: 监控内存使用，超过阈值自动清理缓存

#### 3.1.2 异常恢复
- **进程崩溃**: 自动重启服务，恢复运行状态
- **任务中断**: 支持从断点继续执行，不丢失进度
- **数据损坏**: 数据完整性校验，自动修复或隔离
- **配置错误**: 配置验证，使用默认值或报错提示

#### 3.1.3 降级策略
- **LLM降级**: 主LLM失败时自动切换到备用LLM
- **功能降级**: 非核心功能失败不影响核心流程
- **性能降级**: 高负载时自动降低处理优先级
- **质量降级**: 极端情况下允许降低质量标准完成任务

### 3.2 稳定性要求

#### 3.2.1 运行稳定性
- **服务可用性**: ≥99.5%（月停机时间≤3.6小时）
- **任务成功率**: ≥95%（考虑LLM不稳定因素）
- **数据一致性**: 100%（关键数据不丢失、不冲突）
- **状态一致性**: 100%（任务状态与实际进度一致）

#### 3.2.2 长时间运行
- **7×24运行**: 支持不间断运行，无内存泄漏
- **资源稳定**: 运行72小时后资源占用增长≤10%
- **状态保持**: 服务重启后能恢复所有任务状态
- **日志完整**: 长期运行日志不丢失、不混乱

#### 3.2.3 压力稳定性
- **峰值压力**: 支持短时间内5倍正常负载
- **持续压力**: 支持2倍正常负载持续运行24小时
- **恢复能力**: 压力解除后30分钟内恢复正常性能
- **资源回收**: 压力期后能完全回收临时资源

### 3.3 数据完整性

#### 3.3.1 数据保护
- **任务数据**: 每个任务独立目录，互不干扰
- **进度数据**: 实时保存进度，防止丢失
- **产出文件**: 生成即保存，多副本保护
- **配置数据**: 版本控制，支持回滚

#### 3.3.2 数据一致性
- **状态同步**: 内存状态与存储状态实时同步
- **文件同步**: 文件生成与数据库记录同步
- **进度同步**: 各Agent进度与总进度一致
- **日志同步**: 操作日志与实际操作一致

#### 3.3.3 数据恢复
- **自动备份**: 关键数据每小时自动备份
- **手动恢复**: 支持从备份恢复任务状态
- **增量恢复**: 支持恢复特定时间点的数据
- **验证恢复**: 恢复后自动验证数据完整性

## 6. 监控与可观测性需求

### 6.1 监控指标

#### 6.1.1 性能指标
- **响应时间**: 各API接口的响应时间分布
- **吞吐量**: 单位时间处理的请求数和任务数
- **成功率**: 各操作的成功率和失败率
- **资源使用**: CPU、内存、磁盘、网络使用情况

#### 6.1.2 业务指标
- **任务统计**: 创建、完成、失败的任务数量
- **章节统计**: 生成的章节数量和总字数
- **质量统计**: 审核通过率、修订次数分布
- **效率统计**: 各Agent平均处理时间

#### 6.1.3 健康指标
- **服务状态**: 各服务的运行状态和健康度
- **依赖状态**: 依赖服务（LLM、数据库等）状态
- **队列状态**: 任务队列长度和处理速度
- **错误率**: 各类错误的发生频率和趋势

### 6.2 日志记录

#### 6.2.1 日志级别
- **DEBUG**: 详细调试信息，用于问题排查
- **INFO**: 正常操作信息，用于跟踪流程
- **WARN**: 警告信息，潜在问题但不影响运行
- **ERROR**: 错误信息，操作失败但可恢复
- **CRITICAL**: 严重错误，系统无法正常运行

#### 6.2.2 日志内容
- **操作日志**: 记录所有重要操作和状态变更
- **性能日志**: 记录性能指标和资源使用情况
- **错误日志**: 记录错误详情和上下文信息
- **审计日志**: 记录安全相关操作和访问记录

#### 6.2.3 日志管理
- **结构化日志**: 日志格式统一，便于解析和分析
- **日志轮转**: 自动轮转日志文件，防止过大
- **日志聚合**: 支持将日志聚合到中央存储
- **日志搜索**: 支持按关键词和时间范围搜索日志

### 6.3 告警机制

#### 6.3.1 告警条件
- **性能告警**: 响应时间超过阈值、成功率下降
- **资源告警**: 资源使用率超过阈值、磁盘空间不足
- **错误告警**: 错误率突然升高、连续出现相同错误
- **业务告警**: 任务积压、长时间无新任务完成

#### 6.3.2 告警级别
- **P0紧急**: 系统不可用，需要立即处理
- **P1重要**: 核心功能受影响，需要尽快处理
- **P2警告**: 非核心问题，需要关注和处理
- **P3提示**: 信息性提示，无需立即处理

#### 6.3.3 告警处理
- **多渠道通知**: 支持邮件、即时消息等多种通知方式
- **告警收敛**: 相同告警自动收敛，避免告警风暴
- **告警升级**: 长时间未处理的告警自动升级
- **告警恢复**: 问题解决后自动清除告警

### 6.4 仪表盘

#### 6.4.1 系统仪表盘
- **总体状态**: 系统健康状态、当前负载
- **性能图表**: 响应时间、吞吐量趋势图
- **资源图表**: CPU、内存、磁盘使用情况
- **错误图表**: 错误类型和频率分布

#### 6.4.2 业务仪表盘
- **任务统计**: 任务状态分布、完成趋势
- **章节统计**: 章节生成数量、字数统计
- **质量统计**: 审核通过率、修订次数
- **效率统计**: 各Agent处理效率对比

#### 6.4.3 自定义仪表盘
- **指标选择**: 支持选择关注的指标组合
- **时间范围**: 支持自定义时间范围查看
- **数据导出**: 支持导出图表数据
- **视图保存**: 支持保存自定义视图
